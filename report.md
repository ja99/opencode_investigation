### Detailed Analysis

#### 1. `openrouter.ai` — LLM API (expected)

Requests were `POST https://openrouter.ai/api/chat/completions`. This is the configured provider endpoint.

Note: the path is `/api/chat/completions`, not `/api/v1/chat/completions` as in v1.2.10. The configured `MY_LLM_BASE_URL` is `https://openrouter.ai/api/v1`, so opencode appears to drop the `/v1` segment when constructing the URL. Requests succeeded, so OpenRouter accepts both paths.

#### 2. `registry.npmjs.org` — Runtime npm Installs

opencode dynamically installs provider SDKs at **runtime**. On first startup it fetched **4 packages** (down from 16 in v1.2.10):

| Package | Version | Purpose |
|---|---|---|
| `@opencode-ai/plugin` | 1.3.5 | opencode plugin system |
| `@opencode-ai/sdk` | 1.3.5 | opencode SDK |
| `zod` | 4.1.8 | Schema validation |

The Anthropic auth packages (`opencode-anthropic-auth`, `@openauthjs/openauth`, `arctic`, `jose`, etc.) and the Oslo crypto libraries that were fetched in v1.2.10 are **no longer fetched**.

#### 3. `models.dev` — Model Catalog

`GET https://models.dev/api.json` is fetched **once per prompt** (3 times across the session), compared to once in v1.2.10. Still a third-party service for model metadata.

#### 4. `github.com` + `release-assets.githubusercontent.com` — ripgrep

Same as v1.2.10: `ripgrep-14.1.1-x86_64-unknown-linux-musl.tar.gz` downloaded at runtime from GitHub.

### Key Findings

#### ✅ What's Respected

- **No telemetry endpoints detected** — `openTelemetry: false` is honored
- **No sharing endpoints detected** — `share: "disabled"` is honored
- **No opencode.ai servers contacted** — `disabled_providers: ["opencode"]` is honored
- **No workspace content sent to unexpected destinations**

#### ⚠️ What to Be Aware Of

- opencode still makes **runtime network calls beyond your LLM API** to npm, GitHub, and models.dev
- `models.dev/api.json` is now fetched on every prompt, not just startup
- The LLM API path differs from the configured base URL (`/api/chat/completions` vs `/api/v1/chat/completions`)

### Verdict

| Question | Answer |
|---|---|
| Does it send workspace content to unexpected services? | **No** |
| Does it respect `openTelemetry: false`? | **Yes** |
| Does it respect `share: disabled`? | **Yes** |
| Does it contact services beyond the configured LLM? | **Yes** — npm, GitHub, models.dev |
| Does it contact `opencode.ai` servers? | **No** |
